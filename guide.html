What You'll Need

  • A laptop or desktop with a webcam (built-in or USB — most laptops have one)
  • Python 3.8 or higher installed (python.org/downloads)
  • A text editor or IDE — VS Code, PyCharm, or even IDLE works
  • About 500 MB of disk space for the libraries
  • Patience and curiosity — if something breaks, that's normal. You'll learn the most from debugging.

Everything we use in this guide is free and open-source. You'll install three main Python libraries:

  • OpenCV — captures and processes video from your webcam
  • MediaPipe — Google's machine learning library that detects your hand and its 21 landmarks in real time
  • PyAutoGUI — lets Python move your mouse cursor and simulate clicks

By the end of this guide, you'll have a working gesture-controlled mouse that you built yourself. Let's get started.

Capturing Video from Your Webcam

OpenCV makes it surprisingly easy to open your webcam and start reading video frames. A "frame" is just a single image — video is really a rapid sequence of images (typically 30 per second). Here's the basic pattern you'll use throughout this entire project:

basic_webcam.py 
import cv2

# Open the default webcam (index 0)
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("Error: Could not open webcam!")
    exit()

print("Webcam opened! Press 'q' to quit.")

while True:
    ret, frame = cap.read()

    if not ret:
        print("Error: Can't receive frame.")
        break

    # Show the frame in a window
    cv2.imshow("Webcam Feed", frame)

    # Wait 1ms for a key press; quit on 'q'
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

# Always clean up
cap.release()
cv2.destroyAllWindows()

Run this, and you should see a live video feed from your webcam. If you see yourself on screen, congratulations — you've taken the first step. Press q to close the window.

How cv2.VideoCapture Works

VideoCapture(0) opens your default camera. If you have multiple cameras, try 1, 2, etc. The cap.read() call returns two things: ret (a boolean — did the read succeed?) and frame (the actual image as a NumPy array). Each frame is a 3D array with shape (height, width, 3) where the 3 represents Blue, Green, and Red color channels (yes, BGR, not RGB — OpenCV is quirky like that).

Manipulating Frames

Before we get to hand tracking, let's learn to manipulate video frames. These techniques are tools you'll use throughout the project. Every frame is just a NumPy array, so you can slice, transform, and modify it like any array.

frame_manipulation.py 
import cv2

cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    # FLIP: Mirror the image so it feels natural (like a mirror)
    # flipCode: 0 = vertical, 1 = horizontal, -1 = both
    flipped = cv2.flip(frame, 1)

    # RESIZE: Make the frame smaller (faster processing)
    # You can use exact dimensions or a scale factor
    small = cv2.resize(flipped, (320, 240))

    # Or resize by percentage:
    # half = cv2.resize(flipped, None, fx=0.5, fy=0.5)

    # COLOR CONVERT: Convert BGR to RGB (needed for MediaPipe later)
    rgb = cv2.cvtColor(flipped, cv2.COLOR_BGR2RGB)

    # CROP: Grab just a region of the frame
    # Syntax: frame[y_start:y_end, x_start:x_end]
    h, w = flipped.shape[:2]
    cropped = flipped[0:h//2, 0:w//2]  # Top-left quarter

    # GRAYSCALE: Sometimes useful for image processing
    gray = cv2.cvtColor(flipped, cv2.COLOR_BGR2GRAY)

    # Show different versions
    cv2.imshow("Flipped (Mirror)", flipped)
    cv2.imshow("Small", small)
    cv2.imshow("Cropped", cropped)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()

The most important one for our project is cv2.flip(frame, 1). Without the horizontal flip, when you move your right hand to the right, the cursor moves left on screen. Flipping makes the webcam act like a mirror, which is much more intuitive for gesture control.

Drawing on Frames

You can draw shapes, text, and lines directly onto frames. This is how we'll build a debug overlay later — showing landmark positions, gesture status, and FPS right on the video feed.

drawing_demo.py 
import cv2
import time

cap = cv2.VideoCapture(0)
prev_time = time.time()

while True:
    ret, frame = cap.read()
    if not ret:
        break

    frame = cv2.flip(frame, 1)
    h, w = frame.shape[:2]

    # --- Drawing primitives ---

    # Line: from point A to point B
    cv2.line(frame, (50, 50), (200, 50), (0, 255, 0), 2)

    # Rectangle: top-left corner to bottom-right corner
    cv2.rectangle(frame, (50, 80), (200, 160), (255, 0, 0), 2)

    # Filled rectangle (use -1 for thickness)
    cv2.rectangle(frame, (50, 180), (200, 220), (255, 0, 0), -1)

    # Circle: center point, radius
    cv2.circle(frame, (300, 120), 40, (0, 0, 255), 2)

    # Filled circle
    cv2.circle(frame, (300, 220), 20, (0, 255, 255), -1)

    # --- Text (great for HUD/debug info) ---

    # Calculate FPS
    curr_time = time.time()
    fps = 1 / (curr_time - prev_time) if (curr_time - prev_time) > 0 else 0
    prev_time = curr_time

    # FPS counter in top-right corner
    fps_text = f"FPS: {fps:.0f}"
    cv2.putText(frame, fps_text, (w - 150, 30),
                cv2.FONT_HERSHEY_SIMPLEX, 0.7, (0, 255, 0), 2)

    # Status text with background for readability
    status = "IDLE"
    text_size = cv2.getTextSize(status, cv2.FONT_HERSHEY_SIMPLEX, 0.8, 2)[0]
    cv2.rectangle(frame, (10, h - 50),
                  (10 + text_size[0] + 10, h - 50 + text_size[1] + 10),
                  (0, 0, 0), -1)
    cv2.putText(frame, status, (15, h - 30),
                cv2.FONT_HERSHEY_SIMPLEX, 0.8, (0, 255, 0), 2)

    cv2.imshow("Drawing Demo", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

Notice the FPS counter — you'll want this in your final project to make sure the tracking is running fast enough. Anything above 20 FPS should feel responsive. Below 15, and you'll start to notice lag.

Understanding the Coordinate System

This is one of those things that trips people up if you don't learn it now. In OpenCV's coordinate system, the origin (0, 0) is the top-left corner of the image. The x-axis goes right and the y-axis goes down. This is the opposite of math class, where y usually goes up.

(0,0) ---------> x (width) | | Your webcam frame | (a 2D array of pixels) | v y (height) So a 640x480 frame goes from: Top-left: (0, 0) Top-right: (639, 0) Bottom-left: (0, 479) Bottom-right: (639, 479)

When we get to hand tracking, MediaPipe will give us normalized coordinates (values between 0.0 and 1.0). To convert to pixel coordinates, you multiply: pixel_x = normalized_x * frame_width. Keep this in mind — we'll come back to it.

Check Your Understanding

If your webcam frame is 640x480 and MediaPipe reports a landmark at normalized position (0.5, 0.25), what are the pixel coordinates? Answer: (320, 120) — the center horizontally, a quarter of the way down from the top.

Phase 2: Track Your Hands

This is where it starts to feel like magic. You'll use Google's MediaPipe library to detect and track your hand in real time.

Setting Up MediaPipe Hand Tracking

MediaPipe Hands uses a machine learning model that runs entirely on your CPU — no GPU required. It detects up to two hands and outputs 21 3D landmarks for each one. Here's how to set it up:

hand_tracking_setup.py 
import cv2
import mediapipe as mp

# MediaPipe setup
mp_hands = mp.solutions.hands
mp_draw = mp.solutions.drawing_utils

# Initialize the hand tracker
hands = mp_hands.Hands(
    static_image_mode=False,   # Video mode (faster, uses tracking)
    max_num_hands=1,           # We only need one hand for mouse control
    min_detection_confidence=0.7,  # How sure it must be to detect a hand
    min_tracking_confidence=0.5,   # How sure it must be to keep tracking
)

cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    frame = cv2.flip(frame, 1)

    # IMPORTANT: MediaPipe needs RGB, OpenCV uses BGR
    rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)

    # Process the frame
    results = hands.process(rgb)

    # Draw landmarks if a hand is detected
    if results.multi_hand_landmarks:
        for hand_landmarks in results.multi_hand_landmarks:
            mp_draw.draw_landmarks(
                frame,
                hand_landmarks,
                mp_hands.HAND_CONNECTIONS
            )

    cv2.imshow("Hand Tracking", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
hands.close()

Run this and hold your hand up in front of the webcam. You should see colored dots and lines drawn over your hand. Once you see your hand tracked on screen, it feels like magic — and it's only going to get cooler from here.

Detection Confidence Explained

min_detection_confidence is how sure the model needs to be to detect a new hand (0.0 to 1.0). Higher = fewer false positives but might miss your hand sometimes. min_tracking_confidence is how sure it must be to keep tracking a hand it already found. When tracking fails, it falls back to detection, which is slower. Start with 0.7/0.5 and tweak from there.

Privacy Note

MediaPipe processes everything locally on your machine. No video data is sent to the internet. However, your webcam is active while the program runs. Be aware of your surroundings and close the program when you're done. Consider adding a clear "RECORDING" indicator in your final build.

The 21 Landmarks

MediaPipe identifies 21 specific points on your hand. Understanding which landmark is which is essential for detecting gestures. Here's the map:

MIDDLE (12) RING (16) | INDEX (8) | | | PINKY (20) | | THUMB (4) | \ | | / (19) (15) (11)(7) (3) | | | | | (18) (14) (10)(6) (2) | | | | | (17) (13) (9)(5) (1) \ \ | / / \ \ | / / ------(0)----- <-- WRIST Fingertip IDs: 4 (thumb), 8 (index), 12 (middle), 16 (ring), 20 (pinky) MCP joints (knuckles): 1, 5, 9, 13, 17 PIP joints (middle): 2, 6, 10, 14, 18 DIP joints (upper): 3, 7, 11, 15, 19

Each landmark has three values: x (horizontal position, 0.0 to 1.0), y (vertical position, 0.0 to 1.0), and z (depth — how close to the camera, relative to the wrist). For mouse control, we mostly use the index fingertip (landmark 8) for cursor position and the thumb tip (landmark 4) for detecting pinch gestures.

Here's how to read the coordinates:

# Inside your main loop, after results = hands.process(rgb):

    print(f"Index fingertip: pixel ({ix}, {iy})")

    cv2.circle(frame, (tx, ty), 10, (0, 0, 255), -1)   # Red

    # Draw line between thumb and index (for pinch visualization)
    cv2.line(frame, (tx, ty), (ix, iy), (255, 255, 0), 2)

Detecting Gestures

Now for the fun part. We're going to teach the computer to recognize what your hand is doing. Every gesture boils down to measuring distances and angles between landmarks. Let's start with the most useful gesture for mouse control: the pinch.

Measuring Distances Between Landmarks

distance_helper.py 
import math

def pixel_distance(p1, p2):
    """Calculate distance between two (x, y) points in pixels."""
    return math.sqrt((p1[0] - p2[0]) ** 2 + (p1[1] - p2[1]) ** 2)

Pinch Detection

A pinch is when your thumb tip and index fingertip come close together. We measure the distance between them and compare it to a threshold:

gestures.py — pinch detection 
def is_pinching(hand_landmarks, frame_shape, threshold=40):
    """Detect a pinch gesture (thumb tip close to index fingertip).

    Args:
        hand_landmarks: MediaPipe hand landmarks object.
        frame_shape: (height, width, channels) of the current frame.
        threshold: Max pixel distance to count as a pinch.

    Returns:
        Tuple of (is_pinching: bool, distance: float).
    """
    h, w = frame_shape[:2]

    thumb_tip = hand_landmarks.landmark[4]
    index_tip = hand_landmarks.landmark[8]

    # Convert normalized coords to pixels
    tx, ty = int(thumb_tip.x * w), int(thumb_tip.y * h)
    ix, iy = int(index_tip.x * w), int(index_tip.y * h)

    dist = pixel_distance((tx, ty), (ix, iy))
    return dist < threshold, dist

Finger Extension Detection

To detect if a finger is extended (straight out) or curled, we compare the fingertip's position to its lower joints. If the tip is farther from the wrist than the PIP joint, the finger is probably extended:

gestures.py — finger extension 
def is_finger_extended(hand_landmarks, finger_tip_id, finger_pip_id):
    """Check if a finger is extended (tip is above PIP joint).

    For fingers 8, 12, 16, 20: compare tip.y to pip.y
    (lower y value = higher on screen = extended).
    The thumb (4) needs special handling -- we compare x instead.

    Args:
        hand_landmarks: MediaPipe hand landmarks.
        finger_tip_id: Landmark index of fingertip (4, 8, 12, 16, or 20).
        finger_pip_id: Landmark index of PIP joint (3, 6, 10, 14, or 18).

    Returns:
        True if the finger appears extended.
    """
    tip = hand_landmarks.landmark[finger_tip_id]
    pip_joint = hand_landmarks.landmark[finger_pip_id]

    if finger_tip_id == 4:  # Thumb uses x-axis
        # This works for right hand; for left hand you'd flip the comparison
        return tip.x < pip_joint.x
    else:
        return tip.y < pip_joint.y


def count_extended_fingers(hand_landmarks):
    """Count how many fingers are extended.

    Returns:
        Integer from 0 to 5.
    """
    finger_tips = [4, 8, 12, 16, 20]
    finger_pips = [3, 6, 10, 14, 18]

    count = 0
    for tip_id, pip_id in zip(finger_tips, finger_pips):
        if is_finger_extended(hand_landmarks, tip_id, pip_id):
            count += 1
    return count

Common Gesture Patterns

gestures.py — gesture patterns 
def is_open_hand(hand_landmarks):
    """Detect open hand (all five fingers extended).
    Useful for 'stop' or 'scroll mode' gestures."""
    return count_extended_fingers(hand_landmarks) == 5


def is_pointing(hand_landmarks):
    """Detect pointing gesture (only index finger extended).
    Useful for 'move cursor' mode."""
    tips = [4, 8, 12, 16, 20]
    pips = [3, 6, 10, 14, 18]

    states = []
    for tip_id, pip_id in zip(tips, pips):
        states.append(is_finger_extended(hand_landmarks, tip_id, pip_id))

    # Index extended, others curled (thumb can be either)
    return states[1] and not states[2] and not states[3] and not states[4]

With these building blocks, you can detect quite a few gestures. The pointing gesture (index finger up, others curled) will be your "move cursor" gesture. The pinch will be "click." The open hand could mean "scroll" or "stop." Feel free to invent your own combinations — that's Phase 4.

Smoothing Movement with EMA

If you've tried moving the cursor based on raw hand positions, you've probably noticed it's jittery. Your hand naturally trembles a little, and that gets amplified on a big monitor. The fix is smoothing — specifically, Exponential Moving Average (EMA).

What is EMA?

EMA is a way to average values over time, giving more weight to recent values. The formula is: smoothed = alpha * new_value + (1 - alpha) * previous_smoothed. The alpha parameter (0.0 to 1.0) controls responsiveness. Low alpha (like 0.2) = very smooth but laggy. High alpha (like 0.8) = responsive but jittery. Start with 0.4 and adjust to taste.

smoother.py 
class Smoother:
    """Exponential Moving Average (EMA) smoother for reducing jitter.

    Usage:
        smooth_x = Smoother(alpha=0.4)
        smooth_y = Smoother(alpha=0.4)
        # In your loop:
        sx = smooth_x.update(raw_x)
        sy = smooth_y.update(raw_y)
    """

    def __init__(self, alpha=0.4):
        """
        Args:
            alpha: Smoothing factor (0.0 to 1.0).
                   Lower = smoother but laggier.
                   Higher = more responsive but jittery.
        """
        self.alpha = alpha
        self.value = None

    def update(self, new_value):
        """Feed in a new raw value, get back a smoothed value."""
        if self.value is None:
            self.value = new_value
        else:
            self.value = self.alpha * new_value + (1 - self.alpha) * self.value
        return self.value

    def reset(self):
        """Reset the smoother (e.g., when hand tracking is lost)."""
        self.value = None

You'll create separate smoothers for the x and y coordinates. When tracking is lost (hand leaves the frame), call reset() so the cursor doesn't jump when the hand comes back.

Phase 3: Control the Mouse

Now we connect your hand gestures to actual mouse input. This is where it all comes together.

PyAutoGUI Basics

PyAutoGUI lets Python take control of your mouse and keyboard. It can move the cursor, click, scroll, and type. For our project, we only need cursor movement and clicks.

Important: PyAutoGUI Fail-Safe

PyAutoGUI has a built-in safety feature: if you slam your mouse into any corner of the screen, it raises a pyautogui.FailSafeException and stops the program. Never disable this. When your code is controlling the cursor, this is your emergency stop. If something goes wrong and the cursor is going haywire, just shove your physical mouse to any corner.

pyautogui_basics.py 
import pyautogui
import time

# SAFETY: Keep this enabled! (It's on by default)
pyautogui.FAILSAFE = True

# Disable the default 0.1s pause between actions (we need speed)
pyautogui.PAUSE = 0

# Get screen dimensions
screen_w, screen_h = pyautogui.size()
print(f"Screen size: {screen_w} x {screen_h}")

# Move the mouse to an absolute position
pyautogui.moveTo(screen_w // 2, screen_h // 2)
time.sleep(1)

# Move relative to current position
pyautogui.move(100, 0)   # Move right 100 pixels
time.sleep(0.5)

# Click at current position
pyautogui.click()

# Scroll (positive = up, negative = down)
pyautogui.scroll(3)   # Scroll up 3 "clicks"
time.sleep(0.5)
pyautogui.scroll(-3)  # Scroll down 3 "clicks"

print("Done! (Move mouse to a corner to trigger fail-safe if needed)")
Check Your Understanding

Why do we set pyautogui.PAUSE = 0? Because the default 0.1-second pause between every PyAutoGUI call would make our real-time cursor movement feel sluggish. We need it to move the mouse as fast as our gesture tracking can process frames.

Mapping Hand Position to Screen

Here's the tricky part: your webcam frame is maybe 640x480 pixels, but your screen might be 1920x1080 or larger. You can't just use the hand's pixel position directly — you need to map from webcam space to screen space.

An even better approach is to define a control zone — a smaller area within the webcam frame that maps to the full screen. This means you don't have to wave your hand across the entire frame. Small, comfortable movements control the whole screen.

coordinate_mapping.py 
import pyautogui

def map_to_screen(hand_x, hand_y, frame_w, frame_h, margin=0.15):
    """Map a hand position in webcam space to screen coordinates.

    Uses a 'control zone' defined by the margin parameter.
    The zone is a centered rectangle within the webcam frame.
    Positions inside the zone map to the full screen area.

    Args:
        hand_x: Hand x position in pixels (webcam space).
        hand_y: Hand y position in pixels (webcam space).
        frame_w: Width of webcam frame in pixels.
        frame_h: Height of webcam frame in pixels.
        margin: Fraction of frame to exclude on each side (0.0 to 0.4).

    Returns:
        (screen_x, screen_y) clamped to screen boundaries.
    """
    screen_w, screen_h = pyautogui.size()

    # Define the control zone
    x_min = frame_w * margin
    x_max = frame_w * (1 - margin)
    y_min = frame_h * margin
    y_max = frame_h * (1 - margin)

    # Normalize hand position within the zone (0.0 to 1.0)
    norm_x = (hand_x - x_min) / (x_max - x_min)
    norm_y = (hand_y - y_min) / (y_max - y_min)

    # Clamp to [0.0, 1.0] so cursor stays on screen
    norm_x = max(0.0, min(1.0, norm_x))
    norm_y = max(0.0, min(1.0, norm_y))

    # Scale to screen size
    sx = int(norm_x * screen_w)
    sy = int(norm_y * screen_h)

    return sx, sy

Hard to reach screen corners Margin too small Increase margin to 0.2. The control zone shrinks, making corners easier to reach.

Y-axis feels inverted Coordinate mismatch Both OpenCV and screen coords have y going down, so this shouldn't happen. Check if you accidentally swapped x and y.

Building a Gesture State Machine

Right now, you can detect gestures frame by frame. But what happens when a gesture is detected for just one frame due to noise? You'd get accidental clicks all over the place. The solution is a state machine — a way to manage what mode the system is in and control transitions between modes.

What is a State Machine?

A state machine is a system that can be in exactly one "state" at a time (like IDLE, MOVING, CLICKING) and transitions between states based on conditions. Think of it like a traffic light: it's always in one state (red, yellow, or green), and specific rules control when it changes. For gesture control, the state machine prevents chaotic behavior by enforcing orderly transitions and adding safeguards like cooldowns.

gesture_controller.py 
import time
from enum import Enum

class GestureState(Enum):
    """The possible states of our gesture controller."""
    IDLE = "idle"           # Hand not detected
    MOVING = "moving"       # Index finger up, controlling cursor
    CLICKING = "clicking"   # Pinch detected, performing click
    SCROLLING = "scrolling" # Open hand, scroll mode


class GestureController:
    """Manages gesture states and transitions.

    Prevents accidental clicks with hysteresis (different thresholds
    for activating vs. deactivating) and cooldown timers.
    """

    def __init__(self):
        self.state = GestureState.IDLE
        self.last_click_time = 0
        self.click_cooldown = 0.4  # Seconds between clicks

        # Hysteresis thresholds for pinch detection
        self.pinch_activate = 35    # Distance to start a pinch
        self.pinch_deactivate = 55  # Distance to release a pinch
        # The gap prevents rapid toggling when distance hovers near threshold

    def update(self, hand_detected, is_pinch, pinch_dist,
               is_point, is_open, hand_x, hand_y):
        """Update the gesture state based on current detections.

        Args:
            hand_detected: Boolean, is a hand visible?
            is_pinch: Boolean, are thumb and index close?
            pinch_dist: Float, pixel distance between thumb and index.
            is_point: Boolean, is the pointing gesture active?
            is_open: Boolean, is the hand open?
            hand_x: Float, current hand x position (screen coords).
            hand_y: Float, current hand y position (screen coords).

        Returns:
            The current GestureState after processing.
        """
        if not hand_detected:
            self.state = GestureState.IDLE
            return self.state

        current_time = time.time()

        if self.state == GestureState.IDLE:
            if is_point:
                self.state = GestureState.MOVING
            elif is_open:
                self.state = GestureState.SCROLLING

        elif self.state == GestureState.MOVING:
            # Use hysteresis for pinch transitions
            if pinch_dist < self.pinch_activate:
                if current_time - self.last_click_time > self.click_cooldown:
                    self.state = GestureState.CLICKING
                    self.last_click_time = current_time
            elif is_open:
                self.state = GestureState.SCROLLING
            elif not is_point:
                self.state = GestureState.IDLE

        elif self.state == GestureState.CLICKING:
            # Use the higher threshold to deactivate (hysteresis)
            if pinch_dist > self.pinch_deactivate:
                if is_point:
                    self.state = GestureState.MOVING
                else:
                    self.state = GestureState.IDLE

        elif self.state == GestureState.SCROLLING:
            if not is_open:
                if is_point:
                    self.state = GestureState.MOVING
                else:
                    self.state = GestureState.IDLE

        return self.state

Hysteresis: Why Two Thresholds?

Imagine the pinch threshold is 40 pixels. Your thumb and index are hovering around 38-42 pixels apart. Without hysteresis, the system would rapidly toggle between CLICKING and MOVING on every frame. With hysteresis, you need to go below 35 to start a click and above 55 to end it. This creates a "dead zone" that prevents flickering.

Cooldowns: Preventing Double Clicks

Even with hysteresis, a quick pinch-release-pinch could fire two clicks faster than you intended. The click_cooldown timer (0.4 seconds by default) enforces a minimum gap between click actions. Adjust this to taste — lower for fast double-clicking, higher for more deliberate interaction.

Making It Configurable with JSON

Hard-coding values like smoothing alpha, pinch thresholds, and margins works while you're developing, but you'll want to tweak these without editing code every time. Let's save settings to a JSON file:

settings_manager.py 
import json
import os

DEFAULT_SETTINGS = {
    "smoothing_alpha": 0.4,
    "pinch_activate": 35,
    "pinch_deactivate": 55,
    "click_cooldown": 0.4,
    "margin": 0.15,
    "min_detection_confidence": 0.7,
    "min_tracking_confidence": 0.5,
    "show_debug_overlay": True,
}

SETTINGS_FILE = "settings.json"


def load_settings():
    """Load settings from JSON file, falling back to defaults.

    Returns:
        dict of settings values.
    """
    settings = DEFAULT_SETTINGS.copy()

    if os.path.exists(SETTINGS_FILE):
        try:
            with open(SETTINGS_FILE, "r") as f:
                user_settings = json.load(f)

            # Only apply known keys (ignore junk/old keys)
            for key in DEFAULT_SETTINGS:
                if key in user_settings:
                    settings[key] = user_settings[key]

            print(f"Loaded settings from {SETTINGS_FILE}")
        except (json.JSONDecodeError, IOError) as e:
            print(f"Warning: Could not load {SETTINGS_FILE}: {e}")
            print("Using default settings.")
    else:
        print("No settings file found. Using defaults.")

    return settings


def save_settings(settings):
    """Save current settings to JSON file.

    Args:
        settings: dict of settings to save.
    """
    try:
        with open(SETTINGS_FILE, "w") as f:
            json.dump(settings, f, indent=2)
        print(f"Settings saved to {SETTINGS_FILE}")
    except IOError as e:
        print(f"Error saving settings: {e}")

Now you can tweak settings.json in a text editor and restart the program to see the changes. Later, you could even add keyboard shortcuts to adjust settings on the fly (like pressing + and - to adjust smoothing while the program runs).

Settings File Security

If you ever extend settings to include file paths, API keys, or other sensitive data, validate inputs carefully. Never use eval() on settings values, and sanitize file paths to prevent directory traversal. For this project, our simple numeric/boolean settings are safe, but it's a good habit to build secure defaults from the start.

Combining Everything: The Main Loop

gesture_mouse.py — main loop skeleton 
"""
Gesture-Controlled Mouse
Built following the Mantum Tech build guide.
"""

import cv2
import mediapipe as mp
import pyautogui
import time

# Import our modules (or paste the classes above into this file)
# from smoother import Smoother
# from gestures import is_pinching, is_pointing, is_open_hand
# from gesture_controller import GestureController, GestureState
# from coordinate_mapping import map_to_screen
# from settings_manager import load_settings

def main():
    # --- Setup ---
    settings = load_settings()

    pyautogui.FAILSAFE = True
    pyautogui.PAUSE = 0

    mp_hands = mp.solutions.hands
    mp_draw = mp.solutions.drawing_utils

    hands = mp_hands.Hands(
        static_image_mode=False,
        max_num_hands=1,
        min_detection_confidence=settings["min_detection_confidence"],
        min_tracking_confidence=settings["min_tracking_confidence"],
    )

    cap = cv2.VideoCapture(0)
    if not cap.isOpened():
        print("Error: Cannot open webcam.")
        return

    # State
    controller = GestureController()
    smooth_x = Smoother(alpha=settings["smoothing_alpha"])
    smooth_y = Smoother(alpha=settings["smoothing_alpha"])
    prev_time = time.time()
    prev_scroll_y = None

    print("Gesture Mouse active. Press 'q' to quit.")
    print("Move mouse to screen corner for emergency stop.")

    # --- Main Loop ---
    while True:
        ret, frame = cap.read()
        if not ret:
            break

        frame = cv2.flip(frame, 1)
        h, w = frame.shape[:2]
        rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)

        results = hands.process(rgb)

        hand_detected = results.multi_hand_landmarks is not None

        if hand_detected:
            hand = results.multi_hand_landmarks[0]

            # Get positions
            index_tip = hand.landmark[8]
            ix, iy = int(index_tip.x * w), int(index_tip.y * h)

            # Detect gestures
            pinch, pinch_dist = is_pinching(hand, frame.shape)
            pointing = is_pointing(hand)
            open_hand = is_open_hand(hand)

            # Map to screen coordinates
            sx, sy = map_to_screen(ix, iy, w, h, settings["margin"])

            # Smooth the coordinates
            sx = smooth_x.update(sx)
            sy = smooth_y.update(sy)

            # Update state machine
            state = controller.update(
                True, pinch, pinch_dist, pointing, open_hand, sx, sy
            )

            # Act on the current state
            if state == GestureState.MOVING:
                pyautogui.moveTo(int(sx), int(sy))

            elif state == GestureState.CLICKING:
                pyautogui.click()

            elif state == GestureState.SCROLLING:
                if prev_scroll_y is not None:
                    delta = prev_scroll_y - iy
                    if abs(delta) > 5:
                        pyautogui.scroll(int(delta / 10))
                prev_scroll_y = iy

            # Draw landmarks
            mp_draw.draw_landmarks(frame, hand, mp_hands.HAND_CONNECTIONS)

            # Draw debug overlay (see next section)
            if settings["show_debug_overlay"]:
                draw_debug_overlay(frame, state, pinch_dist, sx, sy, 0)

        else:
            smooth_x.reset()
            smooth_y.reset()
            prev_scroll_y = None
            controller.update(False, False, 0, False, False, 0, 0)

        # FPS calculation
        curr_time = time.time()
        fps = 1 / (curr_time - prev_time) if (curr_time - prev_time) > 0 else 0
        prev_time = curr_time

        cv2.putText(frame, f"FPS: {fps:.0f}", (w - 120, 30),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.7, (0, 255, 0), 2)

        cv2.imshow("Gesture Mouse", frame)

        key = cv2.waitKey(1) & 0xFF
        if key == ord('q'):
            break

    # --- Cleanup ---
    cap.release()
    cv2.destroyAllWindows()
    hands.close()
    print("Gesture Mouse stopped.")


if __name__ == "__main__":
    main()

This is the full loop. Frame in, process, act, display, repeat. Everything you built in Phases 1-3 clicks into place here.

Building a Debug Overlay

debug_overlay.py 
import cv2

# Color map for different states
STATE_COLORS = {
    "idle": (128, 128, 128),     # Gray
    "moving": (0, 255, 0),       # Green
    "clicking": (0, 0, 255),     # Red
    "scrolling": (255, 165, 0),  # Orange
}


def draw_debug_overlay(frame, state, pinch_dist, screen_x, screen_y, fps):
    """Draw a debug information overlay on the frame.

    Shows current state, coordinates, pinch distance, and FPS.

    Args:
        frame: The OpenCV frame to draw on (modified in place).
        state: Current GestureState enum value.
        pinch_dist: Current distance between thumb and index.
        screen_x: Mapped screen X coordinate.
        screen_y: Mapped screen Y coordinate.
        fps: Current frames per second.
    """
    h, w = frame.shape[:2]
    state_name = state.value if hasattr(state, 'value') else str(state)
    color = STATE_COLORS.get(state_name, (255, 255, 255))

    # Semi-transparent background panel
    overlay = frame.copy()
    cv2.rectangle(overlay, (5, h - 130), (260, h - 5), (0, 0, 0), -1)
    cv2.addWeighted(overlay, 0.6, frame, 0.4, 0, frame)

    # State with colored indicator dot
    cv2.circle(frame, (20, h - 110), 6, color, -1)
    cv2.putText(frame, f"State: {state_name.upper()}",
                (35, h - 103), cv2.FONT_HERSHEY_SIMPLEX, 0.55, (255, 255, 255), 1)

    # Pinch distance with bar visualization
    cv2.putText(frame, f"Pinch: {pinch_dist:.0f}px",
                (15, h - 78), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (200, 200, 200), 1)
    bar_width = min(int(pinch_dist), 200)
    bar_color = (0, 255, 0) if pinch_dist > 55 else (0, 0, 255)
    cv2.rectangle(frame, (15, h - 68), (15 + bar_width, h - 60), bar_color, -1)

    # Screen coordinates
    cv2.putText(frame, f"Screen: ({int(screen_x)}, {int(screen_y)})",
                (15, h - 43), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (200, 200, 200), 1)

    # FPS
    cv2.putText(frame, f"FPS: {fps:.0f}",
                (15, h - 18), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (200, 200, 200), 1)
